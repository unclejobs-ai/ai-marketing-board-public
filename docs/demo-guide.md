# Demo Guide

영상에서 보여주기 좋은 흐름입니다.

## 1. 문제부터 보여주기

"대표님들이 막히는 건 글쓰기보다 이번 주에 뭘 먼저 만들지 결정하는 일입니다."

이 메시지로 시작하면 도구 설명보다 문제에 먼저 집중할 수 있습니다.

## 2. 결과물 먼저 보여주기

최종 보고서에서 아래만 짧게 보여줍니다.

- 이번 주 1순위 콘텐츠
- 왜 이 주제를 먼저 해야 하는지
- 유튜브, 블로그, 쇼츠 재활용 방향
- 대표 승인 필요 항목
- 위임 업무지시서

## 3. 입력 자료는 카테고리만 보여주기

실제 고객 문장이나 사업체 자료를 길게 노출하기보다, 어떤 종류의 자료가 필요한지만 보여줍니다.

- 사업 맥락
- 오퍼
- 고객 언어
- 과거 콘텐츠
- 경쟁사 관찰
- 브랜드 톤
- 제작 제약
- 승인 기준

## 4. 직접 따라 해볼 수 있게 연결하기

마지막에는 이렇게 안내합니다.

> 공개 저장소에 입력 워크시트와 Lite Prompt를 올려두었습니다. 먼저 가볍게 따라 해보시고, 더 깊은 프롬프트 구성과 반복 운영 방식은 강의 자료에서 이어서 다루겠습니다.
