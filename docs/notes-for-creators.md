# Notes For Creators

이 자료를 영상, 블로그, 강의 소개에서 보여줄 때의 간단한 가이드입니다.

## 보여주기 좋은 것

- 왜 이 시스템이 필요한지
- 입력 자료를 어떤 관점으로 정리하는지
- 최종 보고서가 대표의 결정을 어떻게 줄여주는지
- 콘텐츠 하나를 여러 채널로 나누는 방식
- 발행 전 리스크를 확인하는 습관

## 너무 오래 보여주지 않아도 되는 것

- 프롬프트를 한 줄씩 설명하는 장면
- 폴더 구조를 오래 훑는 장면
- 내부 운영 규칙을 그대로 읽는 장면
- 실제 고객 사례나 사업체 자료가 식별되는 장면

## 자연스러운 안내 문장

영상이나 글에서는 아래 정도로 안내하면 충분합니다.

> 이 저장소에는 직접 따라 해볼 수 있는 가벼운 템플릿을 올려두었습니다. 더 깊은 프롬프트 구성과 반복 운영 방식은 강의 자료에서 이어서 다룹니다.

핵심은 공개 자료를 보고도 바로 한 번 시도해볼 수 있게 만드는 것입니다. 동시에 실제 운영에서는 더 많은 자료와 검수 기준이 필요하다는 점을 자연스럽게 알려주면 됩니다.
